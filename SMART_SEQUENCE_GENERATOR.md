# 🎬 智能动作序列生成器使用说明（支持组合动作）

## 🌟 功能简介

智能动作序列生成器是一个强大的自动化工具，可以让你：
- ✅ **选择关键动作**：从 13 种预设动作中选择多个作为关键帧
- ✅ **组合动作**：**为同一关键帧添加多个动作**（如：双脚分开 + 抬右手）⭐ 新增
- ✅ **自动生成过渡**：自动创建平滑的中间过渡帧
- ✅ **一键生成**：默认生成 4 拍（4 秒）流畅动画
- ✅ **多种过渡方式**：支持平滑、线性、缓入缓出三种插值方式
- ✅ **动作分类筛选**：按手部、手臂、头部、腿部、腰部快速查找动作 ⭐ 新增

## 🎯 使用步骤

### 步骤 1：打开智能序列生成器

在页面中找到紫色渐变区域：
```
┌─────────────────────────────────────────────┐
│     🎬 智能动作序列生成器                  │
└─────────────────────────────────────────────┘
```

### 步骤 2：（可选）筛选动作类别

使用顶部分类按钮快速查找动作：
```
[全部] [👋 手部] [💪 手臂] [😮 头部] [👣 腿部] [🏃 腰部]
```

点击类别按钮，只显示该类别的动作。

### 步骤 3：选择关键动作并组合

#### 🔥 **核心功能：组合动作**

**示例：创建"双脚分开 + 抬右手"的组合姿势**

方法 1：连续点击同一个动作
```
1. 点击"👣 双脚分开" → 添加到序列
2. 再次点击"👣 双脚分开" → 保持该动作不变
3. 点击"🙋 抬右手" → 叠加到当前帧

结果：
关键帧 1:
  ● 双脚分开
  ● 双脚分开（保持）
  ● 抬右手
  
= 双脚分开 + 抬右手的组合姿势！
```

方法 2：依次添加不同部位的动作
```
1. 点击"👣 双脚分开" → 腿部姿势
2. 点击"🙋 抬右手" → 手部姿势

结果：
关键帧 1:
  ● 双脚分开
  ● 抬右手
  
= 同样是组合姿势！
```

#### 💡 理解组合逻辑

**重要规则**：
- **相同动作连续出现** = 保持该动作不变
- **不同动作连续出现** = 可能是新关键帧
- **同一帧内的所有动作会叠加** = 组合姿势

**示例对比**：
```
序列 A:
关键帧 1: 双脚分开
关键帧 2: 抬右手
→ 两个独立的关键帧，会生成过渡动画

序列 B:
关键帧 1: 双脚分开 + 抬右手
关键帧 2: 双脚伸直 + 左手叉腰
→ 每个关键帧都是组合姿势
```

### 步骤 4：设置参数

在右侧序列预览区，设置以下参数：

#### 拍数选择
- **2 拍**：快速动作（2 秒）
- **4 拍**：标准长度（4 秒）⭐ 推荐
- **8 拍**：较长动画（8 秒）
- **16 拍**：完整表演（16 秒）

#### 过渡方式
- **平滑过渡**：使用余弦插值，最自然 ⭐ 推荐
- **线性插值**：匀速变化，机械感
- **缓入缓出**：慢 - 快-慢节奏，更有动感

### 步骤 5：生成动画

点击 **"✨ 生成 N 拍动画"** 按钮

系统会自动：
1. 清除当前所有帧
2. 为每个关键帧之间生成平滑过渡
3. 创建完整的动画序列
4. 跳转到第 1 帧准备播放

### 步骤 6：预览和调整

- 点击"播放"按钮预览动画
- 如需调整，可以手动修改某一帧的姿势
- 或清空序列重新生成

## 🎨 进阶技巧

### 技巧 1：分层构建组合动作

**思路**：将身体分为不同层次

```
层次结构：
├─ 下半身（腿/脚）- 基础层
├─ 躯干（腰/胸）- 中间层
└─ 上半身（手/头）- 表现层
```

**构建方法**：
```
步骤 1：先选下半身动作
  → 点击"👣 双脚分开"
  
步骤 2：添加躯干动作（可选）
  → 点击"🏃 向左弯腰"
  
步骤 3：添加上半身动作
  → 点击"🤘 左手叉腰"
  → 点击"🙋 抬右手"

最终组合：双脚分开 + 向左弯腰 + 左手叉腰 + 抬右手
```

---

### 技巧 2：保持与变化的艺术

**核心原则**：
- **相同动作连续出现** = 保持该部位不变
- **不同动作出现** = 该部位发生变化

**示例**：
```
序列 A（保持下半身）:
关键帧 1: 双脚分开 + 抬右手
关键帧 2: 双脚分开 + 抬左手
→ 下半身稳定，只有手部变化

序列 B（全身变化）:
关键帧 1: 双脚分开 + 抬右手
关键帧 2: 双脚收齐 + 左手叉腰
→ 全身都在变化
```

**应用场景**：
- **序列 A**：适合局部动作（如挥手、转头）
- **序列 B**：适合全身协调（如舞蹈、体操）

### 技巧 4：循环动画的组合设计

要创建可循环的动画，确保：
1. **起始帧 = 结束帧**（包含所有组合动作）
   ```
   序列：(A+B) → (C+D) → (A+B)
   ```

2. **使用偶数拍**
   - 4 拍、8 拍、16 拍最容易循环

3. **对称组合**
   ```
   关键帧 1: 双脚分开 + 左手叉腰 + 抬右手
   关键帧 2: 双脚分开 + 右手叉腰 + 抬左手
   关键帧 3: 双脚分开 + 左手叉腰 + 抬右手
   
   → 完美的左右对称循环！
   ```

### 技巧 5：复合动作的时间控制

**问题**：组合动作越多，每个动作的时间越短

**解决方案**：
```
方法 1：增加总拍数
  - 从 4 拍增加到 8 拍
  - 每个组合姿势有更多展示时间

方法 2：减少关键帧数量
  - 精简不必要的组合
  - 突出主要动作

方法 3：调整 FPS
  - 降低 FPS：动作更快
  - 提高 FPS：动作更细腻
```

## 🔧 技术原理

### 核心算法升级

#### 1. 组合姿势计算

对于每个关键帧，按顺序应用所有动作：

```javascript
// 伪代码
const targetPose = createDefaultPose();

// 依次应用该帧的所有动作
frameGroup.forEach(actionKey => {
  const action = quickActions[actionKey];
  applyQuickActionToPose(targetPose, action);
});

// 结果：所有动作的叠加效果
```

**叠加规则**：
- 后应用的动作会覆盖先应用的关节
- 不同部位的動作互不干扰
- 相同部位的動作取最后一个

**示例**：
```
动作 1: 双脚分开（修改膝盖、脚踝）
动作 2: 抬右手（修改右肩、右肘、右手腕）

组合结果:
- 膝盖、脚踝 = 双脚分开的值
- 右肩、右肘、右手腕 = 抬右手的值
- 其他关节 = 默认值
```

---

#### 2. 分组显示逻辑

自动识别哪些动作属于同一帧：

```javascript
const groupedSequence = computed(() => {
  const groups = [];
  let currentGroup = [];
  let lastActionKey = null;
  
  sequence.value.forEach((item) => {
    // 如果与前一个动作不同，则开启新组
    if (lastActionKey !== item.actionKey) {
      if (currentGroup.length > 0) {
        groups.push(currentGroup);
      }
      currentGroup = [item.actionKey];
    } else {
      // 相同动作，加入当前组
      currentGroup.push(item.actionKey);
    }
    lastActionKey = item.actionKey;
  });
  
  if (currentGroup.length > 0) {
    groups.push(currentGroup);
  }
  
  return groups;
});
```

**示例**：
```
输入序列：[A, A, B, C, C, C, D]

分组结果:
组 1: [A, A]     → 关键帧 1：动作 A
组 2: [B]        → 关键帧 2：动作 B
组 3: [C, C, C]  → 关键帧 3：动作 C（保持）
组 4: [D]        → 关键帧 4：动作 D
```

---

#### 3. 姿势到姿势的插值

从一个组合姿势过渡到另一个组合姿势：

```javascript
const interpolatePose = (fromPose, toPose, progress) => {
  const newPose = {};
  
  // 复制起始姿势
  for (let jointId in fromPose) {
    newPose[jointId] = { ...fromPose[jointId] };
  }
  
  // 对目标姿势进行插值
  for (let jointId in toPose) {
   if (newPose[jointId]) {
    const fromJoint = fromPose[jointId];
    const toJoint = toPose[jointId];
      
      newPose[jointId].x = fromJoint.x + 
        (toJoint.x - fromJoint.x) * progress;
      newPose[jointId].y = fromJoint.y + 
        (toJoint.y - fromJoint.y) * progress;
    }
  }
  
  return newPose;
};
```

**可视化**：
```
关键帧 1: (A 姿势 + B 姿势) 的组合
  ↓ 插值
中间帧 1: 70% (A+B) +30% (C+D)
  ↓ 插值
中间帧 2: 40% (A+B) + 60% (C+D)
  ↓ 插值
关键帧 2: (C 姿势 + D 姿势) 的组合
```

## 📊 性能优化

### 组合动作的最佳实践

1. **每帧 2-4 个动作为宜**
   - 太少：动作单薄
   - 太多：计算复杂，可能冲突

2. **避免同一关节重复修改**
   ```
   ❌ 不推荐：
   关键帧 1: 抬右手 + 右手伸直
   （两者都修改右臂，会冲突）
   
   ✅ 推荐：
   关键帧 1: 抬右手 + 双脚分开
   （不同部位，互不干扰）
   ```

3. **及时预览检查**
   - 生成后立即播放预览
   - 发现异常及时调整

## 🐛 常见问题

### Q1: 为什么我的组合动作没效果？

**A**: 可能是动作冲突了
```
检查方法：
1. 查看序列列表，是否有修改同一关节的动作
2. 例如："抬右手"和"右手伸直"都修改右臂
3. 解决：只保留其中一个，或选择不冲突的动作
```

### Q2: 如何删除组合中的某个动作？

**A**: 
```
1. 在序列预览区找到该动作
2. 点击动作右侧的"×"按钮
3. 只删除这一个动作，不影响同帧的其他动作
```

### Q3: 如何让某个动作持续多帧？

**A**: 
```
方法 1：连续点击同一个动作
  → 点击"双脚分开"两次 = 保持两帧

方法 2：增加拍数
  → 从 4 拍改为 8 拍，每个姿势持续时间翻倍

方法 3：减少关键帧数量
  → 关键帧越少，每帧时间越长
```

### Q4: 组合动作有数量限制吗？

**A**: 
```
理论上限：无限制
实际建议：每帧 2-4 个动作

原因：
- 太多动作会增加计算复杂度
- 可能导致关节冲突
- 视觉效果过于复杂
```

### Q5: 可以混合使用不同分类的动作吗？

**A**: 当然可以！这正是组合动作的精髓！
```
示例：
腿部（👣 双脚分开）+ 
手部（🤘 左手叉腰）+ 
手臂（💪 右手伸直）+ 
腰部（🏃 向左弯腰）

= 一个复杂的全身协调动作！
```
