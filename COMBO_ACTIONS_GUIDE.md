## ⚡ 快速参考

### 组合公式

| 操作 | 结果 | 说明 |
|------|------|------|
| 点击 A → 点击 B | A+B 组合 | 默认行为 ⭐ |
| 点击 A → 锁定 → 点击 B | A 和 B 独立 | 两个关键帧 |
| 点击 A → 点击 A | A（保持） | 同一个动作持续 |
| 点击 A → 撤销 | 无 | 取消最后操作 |

### Q2: 我想让某个动作单独成为一帧怎么办？

**A**: 有两种方法：

**方法 1**：使用"锁定当前帧"
```
1. 点击动作 A
2. 点击"🔒 锁定当前帧"
3. 点击动作 B

结果：A 和 B 是独立的关键帧
```

**方法 2**：先组合，再手动拆分
```
1. 点击动作 A
2. 点击动作 B（此时 A+B 组合）
3. 在动作 B 上点击"×"删除它
4. 点击"🔒 锁定当前帧"
5. 重新点击动作 B

结果：A 和 B 变成独立的关键帧
```

---

### Q3: 可以组合多少个动作？

**A**: 理论上无限制，但建议：
- ✅ **2-4 个动作**：最佳平衡
- ⚠️ **5-6 个动作**：可能过于复杂
- ❌ **7 个以上**：不推荐，计算复杂且可能冲突

**推荐组合**：
```
腿部（1 个）+ 腰部（0-1 个）+ 手部（1-2 个）= 2-4 个动作
```

---

### Q4: 如果组合的动作有冲突怎么办？

**A**: 后应用的动作会覆盖先应用的关节

**示例**：
```
组合：抬右手 + 右手伸直

结果：
- "抬右手"先应用（修改右肩、右肘）
- "右手伸直"后应用（也修改右臂）
- 最终效果 = 右手伸直的姿态

建议：避免组合修改同一关节的动作
```

## 📈 技术原理

### 自动组合逻辑

```javascript
// 伪代码
const addToSequence = (actionKey) => {
  // 只有第一个动作才创建新帧
  const isNewFrame = sequence.value.length === 0;
  
  // 后续动作默认组合到当前帧
  sequence.value.push({ 
    actionKey, 
    isNewFrame: false  // 默认不创建新帧
  });
};

// 分组显示
const groupedSequence = computed(() => {
  const groups = [];
  let currentGroup = [];
  
  sequence.value.forEach((item) => {
    if (item.isNewFrame) {
      // 开启新帧
      if (currentGroup.length > 0) groups.push(currentGroup);
      currentGroup = [item.actionKey];
    } else {
      // 加入当前帧
      currentGroup.push(item.actionKey);
    }
  });
  
  return groups;
});
```

### 视觉效果

```
输入序列：[A, B, C(锁定), D, E]

分组结果:
组 1: [A, B]      → 关键帧 1: A+B 组合
组 2: [C, D, E]   → 关键帧 2: C+D+E 组合

生成动画时会：
1. 计算 A+B 的组合姿势
2. 计算 C+D+E 的组合姿势
3. 在两个组合姿势之间生成平滑过渡
```
