# 线条小人动画制作器

一个基于 Vue 3 + Vite + FFmpeg.wasm 的网页动画制作工具。

## ✨ 主要功能

- 🎨 **可视化编辑**：拖动关节调整小人姿势
- 📹 **多帧动画**：支持创建多帧动画并预览
- 🎬 **双模式导出**：
  - **FFmpeg 模式**：高质量 MP4，兼容性好（需下载 25MB 核心文件）
  - **快速模式**：使用 WebCodecs/MediaRecorder API，无需下载，秒级导出（现代浏览器）
- 💾 **本地缓存**：FFmpeg 核心文件自动缓存到浏览器，无需重复下载
- ⚡ **快速动作**：一键应用 13 种预设动作（手部动作、手臂动作、头部动作、腿部动作、腰部弯曲、舞蹈姿势）
- 🎬 **智能序列生成器**：选择几个关键动作，自动生成流畅过渡的多帧动画（默认 4 拍，每拍 1 秒）
  - ✨ **组合动作**：为同一关键帧添加多个动作（如：双脚分开 + 抬右手），创建复杂姿势
  - 🏷️ **动作分类**：按手部、手臂、头部、腿部、腰部快速筛选动作

## 🚀 使用说明

### 基本操作

1. **拖动关节**：鼠标点击并拖动小人上的圆点（关节）来调整姿势
2. **添加帧**：调整姿势后点击"+ 添加帧"创建新帧
3. **播放预览**：点击"播放"按钮预览动画效果
4. **快速动作**：点击"⚡ 快速动作"区域的按钮，一键应用预设姿势（13 种动作可选）
5. **智能序列生成**：在"🎬 智能动作序列生成器"中选择 2-5 个关键动作，自动生成流畅的 4 拍动画
   - ✨ **组合动作**：连续点击多个动作，为同一关键帧创建组合姿势（如：双脚分开 + 抬右手）
   - 🏷️ **分类筛选**：使用分类按钮快速找到手部、手臂、头部、腿部、腰部动作
6. **导出视频**：点击"导出 MP4"生成视频文件

### 💡 FFmpeg 本地缓存功能

#### 工作原理

1. **首次使用**：
   - 系统会自动从 CDN 下载 FFmpeg 核心文件（约 25MB）
   - 下载完成后自动保存到浏览器的 IndexedDB
   - 加载完成后即可开始制作动画

2. **后续使用**：
   - 系统会优先从本地缓存加载 FFmpeg
   - 无需重复下载，秒级启动
   - 即使离线也能正常使用导出功能

3. **缓存管理**：
   - FFmpeg 加载成功后，会显示"✅ FFmpeg 已缓存（本地）"提示
   - 点击"🗑️ 清除缓存"按钮可以删除缓存
   - 清除后下次使用会重新下载

#### 优势

- ✅ **节省流量**：只需下载一次，避免重复消耗网络流量
- ✅ **快速启动**：后续使用无需等待下载，直接从本地加载
- ✅ **离线可用**：缓存后可在离线状态下使用导出功能
- ✅ **自动管理**：使用浏览器 IndexedDB 存储，自动管理生命周期

#### 注意事项

- 缓存数据存储在浏览器的 IndexedDB 中
- 清除浏览器数据可能会删除缓存
- 不同浏览器之间的缓存不共享
- 如需强制更新 FFmpeg 版本，可手动清除缓存

## 🛠️ 技术栈

- **前端框架**：Vue 3 (Composition API)
- **构建工具**：Vite 5
- **视频编码**：FFmpeg.wasm 0.12.6
- **数据存储**：IndexedDB（用于缓存 FFmpeg 核心文件）

## 🌐 CDN 源配置

项目配置了多个 CDN 源以提高可用性：

1. unpkg (`https://unpkg.com`)
2. jsDelivr (`https://cdn.jsdelivr.net`)

系统会自动尝试不同的 CDN 源，确保能够成功下载 FFmpeg 核心文件。

## 🔧 缓存实现细节

### IndexedDB 结构

```javascript
Database: 'FFmpegCache'
Version: 1
Store: 'files'
KeyPath: 'name'

// 存储的数据
{
  name: 'ffmpeg-core.js',
  data: <JavaScript 源代码文本>
}
{
  name: 'ffmpeg-core.wasm', 
  data: <WASM 二进制 ArrayBuffer>
}
```

### 加载流程

```
1. 页面加载
   ↓
2. 检查 IndexedDB 缓存
   ↓
3a. 缓存命中 → 从本地加载 → 完成
   ↓
3b. 缓存未命中 → 从 CDN 下载 → 保存到缓存 → 完成
```
