## 💻 技术实现要点

### 1. 腰部弯曲的二维表现

由于是二维平面正对视角，"向左弯腰"实际上是通过以下方式实现：

```javascript
// 关键点：所有上半身关节都向左（x 减小）移动
bendLeft: {
  head: { x: 360, y: 160 },     // 头部左移
  neck: { x: 370, y: 190 },     // 脖子左移
  chest: { x: 380, y: 240 },    // 胸部左移
  hips: { x: 390, y: 300 }      // 臀部微左移
}
```

**视觉原理**：
- x 坐标减小 = 向左移动
- y 坐标保持相对稳定 = 高度基本不变
- 从上到下逐渐减小的偏移量 = 自然的弯曲弧度

### 2. 手臂的协调运动

弯腰时手臂不是僵硬的，而是自然跟随：

```
// 左臂（弯曲侧）
leftShoulder: { x: 350, y: 210 },  // 肩膀左移 + 下沉
leftElbow: { x: 320, y: 260 },     // 手肘更左 + 自然下垂
leftWrist: { x: 300, y: 300 }      // 手腕继续延伸

// 右臂（伸展侧）
rightShoulder: { x: 400, y: 200 }, // 右肩跟随转动
rightElbow: { x: 430, y: 250 },    // 右手肘自然位置
rightWrist: { x: 450, y: 290 }     // 右手腕配合姿态
```

### 3. 腿部动作的对称性

```
// 双脚收齐 - 双腿平行，双脚对齐（与肩同宽）
feetTogether: {
  leftKnee:   { x: 380, y: 400 },  // 左腿位置
  leftAnkle:  { x: 380, y: 490 },  // 左脚踝垂直向下
  rightKnee:  { x: 420, y: 400 },  // 右腿位置
  rightAnkle: { x: 420, y: 490 }   // 右脚踝垂直向下
}
// 注意：这是立正姿势，双腿平行，不是并拢贴在一起

// 双脚打开 - 向两侧展开
feetApart: {
  leftKnee:   { x: 350, y: 400 },  // 左移（远离中心）
  leftAnkle:  { x: 330, y: 490 },  // 左移（大幅远离）
  rightKnee:  { x: 450, y: 400 },  // 右移（远离中心）
  rightAnkle: { x: 470, y: 490 }   // 右移（大幅远离）
}
```

## 🔧 动作设计原则

### 人体工程学考虑

1. **脊柱活动范围**
   - 正常侧弯角度：约 30-45 度
   - 实现方式：每节脊椎移动 10-20 像素
   - 总偏移量：头部相对臀部偏移约 40 像素

2. **腿部活动限制**
   - 膝盖不能向后弯曲
   - 脚踝活动角度合理
   - 双脚最大开度：约 120-140 像素（从中心到一侧 60-70 像素）

3. **手臂协调性**
   - 肩部带动上臂
   - 手肘自然弯曲
   - 手腕灵活但不过度

### 二维平面表现技巧

```javascript
// 正确的二维弯腰理解
// "向左" = x 坐标减小（从观众视角）
// 不是"向前"或"向后"（那是 z 轴）

  // 从上到下逐渐减少左移量，形成弧线

  neck:  { x: 370 },  // 左移 30px
  chest: { x: 380 },  // 左移 20px
```

## 🎯 最佳实践建议

### 1. 动作流畅性

```
✅ 推荐：小幅度多帧
- 弯腰分 3 帧：轻微→中等→最大
- 每帧偏移增加 10-15 像素

❌ 避免：一跳到底
- 不要直接从直立到最大弯曲
- 会显得生硬不自然
```

### 2. 对称动作设计

```
创建对称动画：
帧 1-2: 向左弯腰
帧 3-4: 默认姿势
帧 5-6: 向右弯腰
帧 7-8: 默认姿势

这样节奏感更好！
```

### 3. 复合动作

```
高级用法：组合多个身体部位

例如"指向左下方的舞蹈动作"：
1. 先点击"抬左手"（手臂举起）
2. 再点击"向左弯腰"（身体侧弯）
3. 最后点击"双脚打开"（稳定站姿）

→ 形成完整的舞蹈姿势
```
